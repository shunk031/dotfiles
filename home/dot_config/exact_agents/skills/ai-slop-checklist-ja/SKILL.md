# AI Slop Checklist JA

## 読了時の応答

- この skill を読んだら、`🧾 私は ai-slop-checklist-ja を読みました。` と応答する。

## 概要

日本語の文章を、書き換える前に「AI 臭さ」の観点で点検する。
AI 臭さの中心は、記号や絵文字ではなく「書き手の不在」として扱う。

ユーザーの文章をレビューする前に、詳細基準として [checklist.md](references/checklist.md) を読む。

## ワークフロー

1. 文章の目的、読み手、書き手が引き受けるべき立場を文脈から推定する。
2. [checklist.md](references/checklist.md) を読む。
3. `立場 -> 主体 -> 構造 -> 語彙 -> リズム -> 記号` の順に見る。
4. ユーザーが軽い確認だけを求めていない限り、`checklist.md` の 5 軸で採点する。
5. 影響が大きい順に指摘し、最後に具体的な直し方または短い修正案を出す。

## 出力形式

標準では次の形で返す。

```markdown
## 判定
- 総評: ...
- 書き直し推奨: はい/いいえ
- スコア: ../50

## 重大な順の指摘
- 問題箇所: ...
  理由: ...
  直し方: ...

## チェックリスト
- [ ] 立場
- [ ] 主体
- [ ] 構造
- [ ] 語彙
- [ ] リズム
- [ ] 記号

## 修正案
...
```

短文では圧縮してよいが、`判定`、重要な指摘、`修正案` は残す。

## 注意点

- 体験、数値、固有名詞、失敗、好き嫌い、感情を捏造しない。
- 書き手の立場が足りないときは、書き換える前に不足している材料を示す。
- 事実、意図、必要な丁寧さを保つ。
- 人間らしく見せるために、わざと雑な文章にしない。
- 表層の特徴に寄せすぎない。ダッシュ、カギ括弧、絵文字、Markdown 残骸は、立場と主体を見たあとで直す。
